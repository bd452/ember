# C ABI

Build with `--features capi` (and `fbink` on device if you need the real
renderer from Rust; the current C app handle uses the mock renderer for frame
submit / tap dispatch — see limitations below).

The committed header is [`include/ember.h`](../include/ember.h)
(generated by cbindgen; do not edit by hand). Regenerate after FFI changes:

```sh
cbindgen --config ember/cbindgen.toml \
  --crate ember \
  --output ember/include/ember.h
```

(Exact invocation may match CI; keep the header committed.)

## Conventions

- **Opaque handles** — `ember_*_new` / matching `ember_*_free`.
- **Thread** — all calls on the thread that created `ember_app`.
- **Strings** — UTF-8 `const char*`, copied on the way in.
- **Errors** — failing calls set a thread-local message; read with
  `ember_last_error()` (do not free; valid until the next failing call).
- **Panics** — every entry point uses `catch_unwind`; a Rust panic becomes an
  error / no-op instead of unwinding into foreign frames. Release builds keep
  `panic = "unwind"` for that reason.

## Surface (v1)

| Area | Functions |
|------|-----------|
| Version / errors | `ember_version`, `ember_last_error` |
| `i64` signal | `ember_signal_i64_new/get/set/free` |
| Label | `ember_label_new`, `ember_label_set_text`, `ember_label_bind_i64`, `ember_label_free` |
| Button | `ember_button_new`, `ember_button_on_tap`, `ember_button_free` |
| Stack | `ember_vstack_new`, `ember_hstack_new`, `ember_stack_push_label/button`, `ember_stack_free` |
| App | `ember_app_new`, `ember_app_render_frame`, `ember_app_tap_at`, `ember_app_free` |

`slot`, `for_each`, string signals, and custom components are **not** exposed
yet.

## Minimal C sketch

```c
#include "ember.h"
#include <stdio.h>

static void on_inc(void *user) {
    ember_signal_i64 *sig = user;
    ember_signal_i64_set(sig, ember_signal_i64_get(sig) + 1);
}

int main(void) {
    ember_signal_i64 *count = ember_signal_i64_new(0);
    ember_label *label = ember_label_new("");
    ember_label_bind_i64(label, count, "Count: ", "");

    ember_button *inc = ember_button_new(" + ");
    ember_button_on_tap(inc, on_inc, count);

    ember_stack *root = ember_vstack_new(16);
    ember_stack_push_label(root, label);
    ember_stack_push_button(root, inc);

    ember_app *app = ember_app_new(root, 600, 800); /* consumes root */
    if (!app) {
        fprintf(stderr, "%s\n", ember_last_error());
        return 1;
    }

    ember_app_render_frame(app);
    /* Drive taps yourself, or integrate with your own input loop: */
    ember_app_tap_at(app, 100, 200);
    ember_app_render_frame(app);

    ember_app_free(app);
    ember_button_free(inc);
    ember_label_free(label);
    ember_signal_i64_free(count);
    return 0;
}
```

Link against `libember.so` and ensure the header is on the include path.
The KPM package `com.bd452.ember` installs the `.so` and header for
on-device consumers.

## Ownership notes

- `ember_app_new` **consumes** the root stack (do not `ember_stack_free` it afterward).
- Widgets pushed onto a stack are referenced by the tree; still free the
  handles you created when the app is done, matching the header’s create/free
  pairs (see `ffi.rs` for exact ownership if in doubt).
- Tap callbacks receive the `user_data` pointer you passed to
  `ember_button_on_tap`; the library does not free it.

## Limitations

- No on-device `ember_app_run` yet — C callers drive `render_frame` / `tap_at`
  (or call into a Rust wrapper that owns `FbinkRenderer` + `App::run`).
- Structural UI (`slot` / `for_each`) remains Rust-only.
- Prefer the Rust API for new Kindle apps; use the C ABI when embedding from
  another language.
